React-i18next Migration Batch Prompt

You are a Senior React Engineer and Internationalization (i18n) Specialist.

Context

The legacy translation system currently used in this project is obsolete and scheduled for removal.

We are migrating to react-i18next using the Natural Language Keys approach.

Translation Standard

The original English sentence is the translation key.

Examples:

t("Language")
t("Save Changes")
t("Notification Settings")

The project must maintain:

locales/
├── en.json
├── vi.json

Where:

en.json is the source of truth.
vi.json contains Vietnamese translations.
Both files must contain identical keys.

Required Tasks

For every screen listed above:

Task 1: Extract All User-Facing Text

Scan the entire screen and extract every hardcoded English string, including:

Page titles
Section titles
Labels
Buttons
Menu items
Navigation text
Tabs
Descriptions
Subtitles
Helper text
Placeholder text
Validation messages
Error messages
Success messages
Empty-state messages
Confirmation dialogs
Warning messages
Tooltips
Modal content
Any visible UI text

Do not miss any user-facing text.

Ignore:

Variable names
Function names
Comments
API fields
Internal developer strings
Task 2: Refactor Source Code

Update each screen to use react-i18next.

Requirements:

import { useTranslation } from "react-i18next";

Inside component:

const { t } = useTranslation();

Convert all visible text:

Before:

<h1>Language</h1>
<button>Save Changes</button>

After:

<h1>{t("Language")}</h1>
<button>{t("Save Changes")}</button>

Apply this consistently across the entire screen.

Task 3: Generate Translation Dictionaries

Create a consolidated translation dictionary from ALL processed screens.

Generate:

en.json
{
  "Language": "Language",
  "Save Changes": "Save Changes"
}
vi.json
{
  "Language": "Ngôn ngữ",
  "Save Changes": "Lưu thay đổi"
}

Requirements:

Deduplicate keys globally.
Use the exact English text as the key.
Preserve capitalization and punctuation.
Ensure en.json and vi.json contain identical key sets.
Return valid JSON only.
Use professional Vietnamese translations.
Migration Rules
Treat the old translation mechanism as deprecated.
Do not preserve legacy translation helpers.
Do not modify business logic.
Do not change component behavior.
Do not alter UI layout or styling.
Do not rename variables, functions, interfaces, or props.
Preserve all TypeScript types.
Only internationalize user-facing text.
If text is already wrapped in t(), leave it unchanged.
The result must be production-ready.
